Enfocado en asegurar que los datos de diferentes aplicaciones y repositorios de datos sea consistencte. Retos

Inconsistencia Temporal

Ocurre cuando los datos son actualizados en el sistema origen pero no se ha propagado y sincronizado con todos los sistemas destino. Se presenta como consecuencia o costo por mejorar la disponiblidad y rendimiento. Los patrones de integración centrado en aplicación estan basado en APIs y pueden ofrecer fuerte consistencia temporal de datos usando comunicación asincrona entre el consumidor y proveedor. Patrones centrado en eventos pueden relaja el acoplamiento entre los participantes pero incrementa la inconsistencia temporal mientras los datos cambian y se propagan.

Inconsistencia Estructural

Ocurre cuando los datos son representados en diferentes formatos o esquemas. Aplica para datos estructurados y no estructurados.

Inconsistencia Semantica

Ocurre cuando los sistemas involucrados interpretan el significado de los datos de forma diferente.
